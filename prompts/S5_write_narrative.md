1. `tests/baseline_2026-06/06_narrative/{CODE}.json` 或同目录任一文件 —— **黄金范例**：输出格式、语气、颗粒度全部向它对齐。

3. `{PERIOD_DIR}/04_research/研究_{RESEARCH_ID}.md` —— 本月事实素材（唯一的新闻来源，不要凭记忆补充）。
4. `{PERIOD_DIR}/06_narrative/_tokens.txt` —— 本期可用的数字占位符表及其当前值。

## 写出 `{PERIOD_DIR}/06_narrative/{CODE}.json`

字段：`code / title / subtitle / timeline[] / why_html / forward_html / predictions[]`（可选 `background_html`）。

- `timeline`: 4-8 条本月大事，{"date":"6/17","text":"..."}，从研究笔记里挑对股价最重要的，按时间序。
- `why_html`: 一段 `<p>`，回答「本月为什么涨/跌成这样」——因果链条完整、落在事实上，重要判断用 `<b>` 标注。
- `forward_html`: 一段 `<p>`，回答「往前看怎么办」——给出明确立场与边界条件（什么情况该拿住/加/减），不许「一方面另一方面」和稀泥。
- `predictions`: 1-3 条，{"text":"...","window":"..."}。**必须可证伪**：写清验证标准和时间窗口，下期要逐条打分的。

## 硬性规则（违反任何一条会被校验器打回重写）

1. **数字占位符**：凡 `_tokens.txt` 里有的数字（本标的的涨跌幅/盈亏/市值/份额/成本/汇率/基准指数），正文一律写 `{{token}}`，不许手写数值。研究笔记里的数字（如"回购90亿"）可以直接写，但必须带日期语境。
   **跨标的断言禁手写**：持仓排名一律用 `{{h.{CODE}.rank}}`（如"组合{{h.{CODE}.rank}}持仓"），绝不自己判断"第一大/最大持仓"；"组合里唯一…"这类横向比较，必须先对照 `_tokens.txt` 里全部标的的数值核实后才可写。
2. **说人话**：不留没拆开的术语。写「除净日（从这天起买入拿不到这笔分红，股价自动扣掉股息）」，不写裸的「除净日」；类比只用日常事物。
3. 观点改变必须显式说明依据；与上期档案立场的关系要在 forward_html 里体现。
4. 语气：像懂行的朋友，直接、具体、有立场；禁止「仅供参考」式套话（页脚已有免责声明）。
5. 只写这一个 JSON 文件，不改其他文件。

完成后只输出一行：`S5 {CODE} done`。

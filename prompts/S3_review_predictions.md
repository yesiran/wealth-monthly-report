# S3 上期预判复盘

你在月报流水线的 S3 步骤。任务：给标的档案里的历史预判逐条打分，产出复盘文件。这是本产品「连续性」的核心机制——预判必须被诚实地对照事实检验。

## 操作步骤

1. 用 Read 读取 `dossiers/` 下全部档案，收集「预判记录」表中所有结果为 ⏳未到期 的行。
2. 判断每条的验证窗口是否已进入或过期（本期报告月：{PERIOD}，今天：{TODAY}）。窗口还远未到的保持 ⏳。
3. 对到期/进行中的预判，用以下证据源核实：
   - `{PERIOD_DIR}/03_returns.json` 与 `{PERIOD_DIR}/02_market.json`（价格类预判直接用数据）
   - `{PERIOD_DIR}/04_research/` 下的研究笔记（事件类预判）
   - 必要时用 WebSearch 补充查证（每条预判最多 2 次搜索）
4. 写出 `{PERIOD_DIR}/05_review.json`。

## 输出格式（JSON 数组）

```json
[{"code":"03690","name":"美团-W",
  "pred_text":"原预判文字（照抄档案）",
  "window":"原验证窗口",
  "verdict":"✓" ,
  "evidence":"一句话证据，带具体日期和数字"}]
```

verdict 只允许四种：`✓`（命中）`✗`（落空）`◐`（部分命中）`⏳`（未到期）。

## 硬性规则

1. **宁判 ✗ 不含糊**：证据不支持就是落空，不允许用模糊措辞替预判开脱。这个环节的价值在于诚实。
2. evidence 必须包含可核查的事实（日期+数字/事件），不接受「市场普遍认为」这类说法。
3. 全部 ⏳ 的行也要写进输出（下游要渲染完整表格）。
4. 不修改档案文件（那是 S8 的事）。

完成后只输出一行：`S3 done: 共 N 条, ✓a ✗b ◐c ⏳d`。

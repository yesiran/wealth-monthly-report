# S8 档案回写（报告定稿后执行）

你在月报流水线的最后一步，把本期（{PERIOD}）的结论沉淀进标的档案，供下期使用。这是「连续性」的存档动作，认真做。

## 对每一个标的（`{PERIOD_DIR}/06_narrative/` 下每个 <code>.json）

用 Read 读取该标的的叙事 JSON、`dossiers/<code>.md`、`{PERIOD_DIR}/05_review.json`（若存在），然后用 Edit 更新 `dossiers/<code>.md`：

1. **预判记录表**：
   - 把 05_review.json 里该标的的判定结果（✓/✗/◐）与证据填入对应旧行的「结果/复盘证据」列；
   - 把本期叙事 predictions 追加为新行（提出期={PERIOD}，结果=⏳未到期）。
2. **大事记档案**：新增 `### {PERIOD}` 小节，内容=本期叙事 timeline（每条一行，去掉 HTML 标签）。
3. **持仓论点与关键假设**：
   - 若本期复盘/研究证伪了某条关键假设 → 修订论点与该假设，并在「观点变化日志」追加一行：`- {PERIOD}: 假设「XX」被证伪（证据…），论点由…调整为…`；
   - 若没有假设被证伪 → 论点原文**一字不动**，日志追加一行 `- {PERIOD}: 论点维持。本月要点：（一句话）`。
4. frontmatter 的 `updated` 改为 {TODAY}。

## 新标的 / 清仓标的

- 本期新出现的持仓（dossiers/ 下没有档案）：按现有档案的完整结构新建，论点从本期叙事 forward_html 提炼。
- 本期消失的持仓：档案不删，frontmatter 加 `status: closed`，日志记一行清仓观察。

## 硬性规则

1. 观点漂移是这个系统的头号敌人：**没有证伪证据就不许改论点**。
2. 只允许结构化追加与指定字段修订，不重写历史记录。
3. 全部完成后输出一行：`S8 done: 更新 N 份档案, 论点变更 M 份`。
